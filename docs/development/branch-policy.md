# Политика жизненного цикла веток

> ARCH-115. Цель — не накапливать stale-ветки: на момент аудита их было ≥15 локально
> (включая 2-недельный `feat/arch-015-rows-wip`), что создаёт риск конфликтов с забытым WIP.

## Именование

| Префикс | Когда | Пример |
|---|---|---|
| `feat/<slug>` | новая функциональность | `feat/twa-executor-transfers` |
| `fix/<id>-<slug>` | багфикс (с ID из бэклога, если есть) | `fix/fs06-adhoc-keyboard-details` |
| `hotfix/<slug>` | срочный прод-фикс вне очереди | `hotfix/login-500` |
| `chore/<slug>` | инфраструктура, зависимости, доки | `chore/reg02-03-cr-followups` |
| `docs/<slug>` | только документация | `docs/pr37-docs-bundle` |

- Один PR — одна ветка — одна логическая задача. Имя стабильно от создания до мержа.
- `worktree-agent-*` — служебные ветки git-worktree агентов; считаются эфемерными (см. ниже).

## Хранение (retention)

| Состояние | Срок | Действие |
|---|---|---|
| Смержена в `main` | сразу после мержа | удалить (`--delete-branch` при merge или вручную) |
| Не смержена, активна | пока идёт работа | оставить |
| Не смержена, без коммитов > 30 дней | 30 дней | пометить tag-архивом `archive/<branch>-<YYYYMMDD>`, затем удалить |
| `worktree-agent-*`, без коммитов > 7 дней | 7 дней | удалить (worktree auto-cleanup) |

**Целевой инвариант (AC):** число локальных/remote feature-веток ≈ (активные in-flight задачи) + `main`.
Проверка:

```bash
git for-each-ref --sort=-committerdate \
  --format='%(refname:short)  %(committerdate:relative)' refs/heads/
```

## Архивация перед удалением (не теряем работу)

Прежде чем удалять не-смерженную ветку, сохранить указатель тегом:

```bash
git tag archive/<branch>-$(date +%Y%m%d) <branch>
git push origin archive/<branch>-$(date +%Y%m%d)
git branch -D <branch>           # деструктивно — только после подтверждения владельца
git push origin --delete <branch>
```

> Деструктивные git-операции (`branch -D`, удаление remote-веток, force push)
> выполняются только по явному подтверждению владельца репозитория.

## Stale-branch reaper (информационный)

Еженедельный CI-отчёт `.github/workflows/stale-branches.yml` перечисляет remote-ветки
без коммитов > 30 дней в job summary. **Только отчёт, без авто-удаления** —
решение об удалении остаётся за владельцем (репозиторий ведёт один разработчик).

## Ретроспективная чистка

При первом применении: прогнать проверку выше, по каждой ветке старше 30 дней —
смержена ли (если да → удалить), нужна ли (если нет → архив-тег + удалить).
`feat/arch-015-rows-wip` и подобные исторические WIP — кандидаты на архивацию.
