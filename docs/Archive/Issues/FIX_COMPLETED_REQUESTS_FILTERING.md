# Исправление: Фильтрация исполненных заявок

**Дата**: 16 октября 2025
**Статус**: ✅ Исправлено

## Проблема

В меню менеджера "✅ Исполненные заявки" → "📋 Все исполненные" отображались заявки, которые менеджер уже подтвердил и отправил на приёмку заявителю. Эти заявки должны быть в разделе "⏳ Не принятые".

### Симптомы:
- В "📋 Все исполненные" показываются заявки с `manager_confirmed = true`
- Заявки дублируются между разделами "Все исполненные" и "Не принятые"
- Запутывающая логика: что значит "все исполненные"?

## Причина

### Неправильная логика фильтрации

**Было** ([admin.py:1055](uk_management_bot/handlers/admin.py#L1055)):
```python
# Все исполненные заявки: статус "Выполнена"
q = (
    db.query(Request)
    .filter(Request.status == "Выполнена")  # ❌ БЕЗ проверки manager_confirmed
    .order_by(...)
)
```

**Результат**: Показывались ВСЕ заявки со статусом "Выполнена", включая те, что уже подтверждены менеджером.

### Жизненный цикл заявки:

```
"Новая" → "В работе" → "Выполнена" → "Подтверждена" → "Принято"
                             ↓
                  manager_confirmed = false
                  (ожидает проверки менеджером)
                             ↓
                  Менеджер проверяет и подтверждает
                             ↓
                  manager_confirmed = true
                  status = "Выполнена"
                  (ожидает приёмки заявителем)
                             ↓
                  Заявитель принимает
                             ↓
                  status = "Принято"
```

## Решение

### 1. Обновлена фильтрация "📋 Все исполненные"

**После** ([admin.py:1056-1059](uk_management_bot/handlers/admin.py#L1056)):
```python
# Все исполненные заявки: статус "Выполнена" и НЕ подтверждены менеджером
# (ожидают проверки и подтверждения менеджером)
q = (
    db.query(Request)
    .filter(
        Request.status == "Выполнена",
        Request.manager_confirmed == False  # ✅ Только НЕподтверждённые
    )
    .order_by(...)
)
```

### 2. Обновлена статистика

**Было** ([admin.py:1016](uk_management_bot/handlers/admin.py#L1016)):

## Логика разделов

### 📋 Ожидают проверки
**Условие**: `status = 'Выполнена' AND manager_confirmed = false`

Заявки, которые:
- ✅ Исполнитель отметил как "Выполнена"
- ❌ Менеджер ещё НЕ проверил

**Действия менеджера**:
- Проверить выполнение
- Подтвердить → переходит в "Не принятые"
- ИЛИ Вернуть исполнителю → остаётся в "Возвращённые"

### 🔄 Возвращённые
**Условие**: `status = 'Выполнена' AND is_returned = true AND manager_confirmed = false`

Заявки, которые:
- ✅ Менеджер вернул исполнителю на доработку
- ❌ Исполнитель ещё НЕ исправил

### ⏳ Не принятые (заявителем)
**Условие**: `status = 'Выполнена' AND manager_confirmed = true AND is_returned = false`

Заявки, которые:
- ✅ Менеджер подтвердил
- ✅ Отправлены заявителю на приёмку
- ❌ Заявитель ещё НЕ принял

**Действия**:
- Ждать, пока заявитель примет
- Или эскалировать, если долго не принимают

## Тестирование

### Проверено:

```sql
-- ДО исправления:
SELECT count(*) FROM requests
WHERE status = 'Выполнена';
-- Результат: 2 (обе с manager_confirmed = true)

-- ПОСЛЕ исправления:
-- "Ожидают проверки"

-- Результат: 0
```

## Сценарии использования

### Сценарий 1: Исполнитель выполнил заявку
1. Исполнитель: "Отметить как выполнена"
2. `status = 'Выполнена'`, `manager_confirmed = false`
3. **Заявка появляется**: "📋 Ожидают проверки"
4. Менеджер: проверяет и подтверждает
5. `manager_confirmed = true`
6. **Заявка перемещается**: "⏳ Не принятые"
7. Заявитель: принимает работу
8. `status = 'Принято'`
9. **Заявка исчезает** из всех разделов (архив)

### Сценарий 2: Менеджер вернул на доработку
1. Исполнитель: "Отметить как выполнена"
2. `status = 'Выполнена'`, `manager_confirmed = false`
3. **Заявка появляется**: "📋 Ожидают проверки"
4. Менеджер: возвращает на доработку
5. `is_returned = true`
6. **Заявка перемещается**: "🔄 Возвращённые"
7. Исполнитель: исправляет и снова отмечает выполненной
8. `is_returned = false`
9. **Заявка возвращается**: "📋 Ожидают проверки"
