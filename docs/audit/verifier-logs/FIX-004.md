### Bugfix found during execution

**Runbook used reserved name `pg_admin`** — PostgreSQL rejects `CREATE ROLE pg_admin` with `role name "pg_admin" is reserved` (the `pg_*` prefix is reserved for system roles). Runbook patched in-place: `pg_admin` → `uk_admin` everywhere (35 occurrences in runbook + ARCH-108 in backlog + execution-report). Verifier-log self-references also updated.
