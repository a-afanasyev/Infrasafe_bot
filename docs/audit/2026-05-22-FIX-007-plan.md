# FIX-007 — Inbound webhook receiver с HMAC verification — план

**Backlog:** `docs/audit/2026-05-20-backlog.md` → FIX-007 (P0, ungated после ARCH-014).
**Контракт:** `docs/superpowers/specs/2026-05-14-uk-infrasafe-web-integration-plan.md` §4.4.
**Точка отсчёта:** main `732e365`.

## Контекст

`UK_WEBHOOK_SECRET` / `UK_WEBHOOK_SECRET_NEXT` объявлены в `settings.py:103-104`, но
endpoint-приёмник от InfraSafe не реализован — обращение даёт 404, поверхность атаки = 0.
FIX-007 — gate: inbound endpoint должен **с первого коммита** включать HMAC-проверку,
иначе появляется окно без защиты.

**Направление:** InfraSafe → UK. InfraSafe шлёт алерт → UK создаёт заявку.
- **Sender:** InfraSafe `src/clients/ukApiClient.js`, подписывает `UK_WEBHOOK_SECRET`
  (или `_NEXT` при `UK_USE_NEXT_SECRET=true`).
- **Verifier:** UK — реализуем здесь. Принимает **оба** секрета (OLD ‖ NEW), без флага.

**HMAC-схема** (зеркало UK-outbound `webhook_sender.sign_payload`):
- Заголовок `x-webhook-signature: t=<unix>,v1=<hmac_hex>`
- Подпись: `HMAC-SHA256(secret, f"{t}.{raw_body}")`, hex
- Replay-окно: `|now - t| ≤ 300s`

## Scope — две фазы

| Фаза | Что | В FIX-007? |
|---|---|---|
| **Фаза 1 — Security envelope** | HMAC verify, timestamp window, replay-dedup, rate-limit, audit-log, endpoint | ✅ **это и есть FIX-007** (AC ровно про это) |
| **Фаза 2 — Handler alert→request** | Маппинг алерта InfraSafe в `Request`, создание заявки | ⏭ отдельный ticket — нужен подтверждённый payload от InfraSafe-оператора |

Фаза 1 закрывает AC бэклога (401/401/409). Фаза 2 — бизнес-логика, зависит от
координации формата payload; в Фазе 1 handler — валидирующий stub, возвращает 202 Accepted.

> **⚠️ Взаимодействие с ARCH-113 (заведено 2026-05-22).** ARCH-113 унифицирует
> исходящий emit `request.*` так, что **любое** создание заявки шлёт webhook в
> InfraSafe. Заявка, созданная Фазой 2 из InfraSafe-алерта, **не должна**
> эмитить `request.created` обратно в InfraSafe — это echo-loop. Фаза 2 обязана
> создавать такую заявку по пути, который НЕ триггерит outbound-emit (флаг
> `source="infrasafe"` / `suppress_webhook=True` в request-core, либо отдельная
> ветка). Этот инвариант — общий контракт FIX-007 Фаза 2 ↔ ARCH-113;
> зафиксировать при реализации любого из них первым.

## Архитектура (Фаза 1)

```
POST /api/v2/webhooks/infrasafe/alert
        │
        ├─ @limiter.limit("60/minute")          ← slowapi, client_ip_key
        │
        ├─ raw = await request.body()           ← СЫРЫЕ байты до JSON-парсинга
        │
        ├─ verify_signature(raw, header, [SECRET, SECRET_NEXT])
        │     ├─ нет заголовка / битый формат      → 401
        │     ├─ |now - t| > 300s                  → 401  (stale)
        │     └─ ни один секрет не сошёлся          → 401  (constant-time compare)
        │
        ├─ payload = InfrasafeAlertIn.model_validate_json(raw)
        │     └─ schema invalid                     → 422
        │
        ├─ replay check: Redis SET NX webhook:nonce:{event_id} EX 600
        │     └─ ключ уже есть                      → 409  (replay/duplicate)
        │
        ├─ audit log (event_id, ip, result)
        │
        └─ Фаза 1: return 202 Accepted
           Фаза 2: create Request → return 202 + request_number
```

## Решения

- **D1 — Путь endpoint'а.** `POST /api/v2/webhooks/infrasafe/alert`. Все роутеры UK
  под `/api/v2/`; proxy срезает `/uk`. **Координация:** URL должен совпадать с тем, что
  шлёт InfraSafe `ukApiClient.js` — согласовать с оператором до deploy.
- **D2 — Raw body.** HMAC считается над сырыми байтами. Endpoint принимает `Request`,
  делает `await request.body()` ДО парсинга, парсит JSON вручную
  (`model_validate_json(raw)`). Нельзя брать тело как Pydantic-аргумент — re-serialization
  изменит байты и подпись не сойдётся.
- **D3 — Replay-dedup.** Redis `SET webhook:nonce:{event_id} 1 NX EX 600`. `event_id`
  из payload (InfraSafe шлёт его как UK). TTL 600s = 2× replay-окно — за пределами окна
  timestamp-проверка и так отбьёт. Redis-клиент — переиспользуем `redis.asyncio`
  (как `redis_pubsub.get_pubsub_redis`, но отдельный ключ-namespace).
  Redis недоступен → **fail-closed на dedup, fail-open на запрос**: логируем warning,
  пропускаем (HMAC уже проверен — двойная доставка безопаснее потери; InfraSafe-sender
  идемпотентен по своей природе). Альтернатива fail-closed обсуждаема — см. open questions.
- **D4 — Dual-secret.** Verifier перебирает `[UK_WEBHOOK_SECRET, UK_WEBHOOK_SECRET_NEXT]`,
  принимает любой совпавший. Пустые секреты пропускаются. Флаг `_USE_NEXT` для verifier'а
  НЕ нужен (он только sender-side).
- **D5 — Misconfiguration.** Если `UK_WEBHOOK_SECRET` пуст (и `_NEXT` пуст) — endpoint
  отвечает **503** + `logger.error`, НЕ 200. Запуск без секрета = сломанная интеграция,
  а не «всё ок».
- **D6 — Audit.** Фаза 1 — структурированный `logger.info`/`logger.warning`
  (event_id, source_ip, outcome=accepted|rejected_*, secret_slot=primary|next).
  Персистентная таблица `webhook_inbox` (durable audit + idempotency за пределами
  Redis-TTL) — **отдельный follow-up ticket**, не раздуваем P0 до миграции.
- **D7 — Коды ответов.** no/bad sig → **401**; stale timestamp → **401**;
  schema invalid → **422**; replay → **409**; no secret configured → **503**;
  rate-limit → **429**; ok → **202 Accepted**. InfraSafe-sender обязан трактовать
  409 как success (idempotent re-delivery).

## verify_signature — эскиз

```python
# api/webhooks/security.py
import hmac, hashlib, time

def verify_signature(raw_body: bytes, header: str | None,
                     secrets: list[str], window_sec: int = 300) -> tuple[bool, str]:
    """(ok, reason). reason ∈ {'', 'no_header', 'bad_format', 'stale', 'no_match'}."""
    if not header:
        return False, "no_header"
    try:
        parts = dict(p.split("=", 1) for p in header.split(","))
        ts, sig = parts["t"], parts["v1"]
        ts_int = int(ts)
    except (ValueError, KeyError):
        return False, "bad_format"
    if abs(int(time.time()) - ts_int) > window_sec:
        return False, "stale"
    message = f"{ts}.".encode() + raw_body
    for secret in secrets:
        if not secret:
            continue
        expected = hmac.new(secret.encode(), message, hashlib.sha256).hexdigest()
        if hmac.compare_digest(expected, sig):
            return True, ""
    return False, "no_match"
```

> **Контракт с InfraSafe.** `message` = `f"{t}." + raw_body`. UK-outbound `sign_payload`
> кодирует `f"{timestamp}.{body}"` где `body` — JSON-строка. InfraSafe-sender обязан
> подписывать так же: `t` + `.` + ровно те байты тела, что уходят в HTTP. До deploy —
> кросс-проверить с `ukApiClient.js` (R-18 в integration-plan).
