#### FIX-009 — `verify_password` пробрасывает `bcrypt.ValueError` как 500 — ✅ closed
- **Priority:** P3 — **Type:** FIX / SEC — **Source:** найдено при INT-120 prod-smoke 2026-05-25
- **Files:** `uk_management_bot/api/auth/service.py:40-50`, `tests/api/test_auth_service.py`
- **Description:** `bcrypt.checkpw(plain.encode(), hashed.encode())` бросает `ValueError("Invalid salt")` если в БД лежит malformed hash (стрипнут shell-escape'ом `$`, обрезан, не bcrypt-формат). Login-handler в `auth/router.py:153-159` не оборачивает это в try/except — Starlette преобразует unhandled exception в HTTP 500 вместо ожидаемого 401 «Invalid credentials». Любая corruption в `users.password_hash` (manual edit, bad seed migration, backup/restore drift) даёт 500 вместо понятной ошибки. Воспроизведено на live проде: при INT-120 UI-smoke `psql -c "UPDATE users SET password_hash = '$2b$...'"` через SSH стрипнул `$` → в DB ушёл `b2.WVBA.no.prefix...` (51 char вместо 60) → каждый POST `/login` 500'ил с `ValueError: Invalid salt`.
- **Fix:** ✅ Закоммичено (commit `3b0ab52`, prod-deployed) — `verify_password` оборачивает `bcrypt.checkpw` в `try/except (ValueError, TypeError) → return False`. Failed bcrypt → failed credential, никогда не server error. +3 тест-кейса (`test_verify_password_handles_malformed_hash`: no prefix, empty, garbage). Regression 350/350 на tests/api + tests/services/test_reconciliation.py.
