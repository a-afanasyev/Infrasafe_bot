## Закрыто (PR-1…PR-5, код в рабочем дереве, не задеплоено)

- **CODE-1 / CODE-12** — approve/reject/delete сотрудника перерисовывали список через `show_employee_list(callback)`, который парсил `callback.data` не того формата → IndexError, карточка stale. Фикс: approve/reject → карточка через `_return_to_employee_info` (как block/unblock) с `language`; delete → нейтральный экран + кнопка в `employee_mgmt_list_pending_1`. `handlers/employee_management.py`, `keyboards/employee_management.py`.
- **APIFE-1** — `POST /executor/shifts/start` допускает несколько active-смен (намеренно, мульти-специализация), но 3 читателя падали `MultipleResultsFound`→500. Фикс: `.order_by(start_time desc, id desc).limit(1).scalars().first()`. `api/shifts/executor_router.py`, `api/shifts/service.py`.
- **APIFE-9/10** — `list_requests` offset<0 / limit=0 → 500 (Postgres); распаковка None после workflow-перехода → TypeError→500. Фикс: `offset ge=0`, `limit ge=1`, None-guard→404. `api/requests/router.py`.
- **F-02** — refresh-token ротировался неатомарно (гонка double-issue). Фикс: `_rotate_refresh_token` — `SELECT ... FOR UPDATE` + re-check под локом + revoke/issue в одном commit. Доказано PG-тестом. Клиент: кросс-таб координация (Web Locks + localStorage-lock fallback + маркер `uk_refreshed_at`). `api/auth/router.py`, `frontend/src/api/client.ts`.
- **APIFE-14 / F-02 family** — token-family reuse detection: миграция `003` (`family_id`, `parent_token_id`, `revocation_reason`), fail-closed отзыв всей family при предъявлении ротированного токена. `api/auth/router.py`, `database/models/refresh_token.py`, `alembic/versions/0003_*`.

## Open / deferred

- **APIFE-5 (P2, open)** — `find_overlapping_shift_for_update` (`api/shifts/service.py`) из-за предиката `Shift.end_time > start_time` **НЕ видит open-ended смены** (`end_time IS NULL`), которые создаёт executor `POST /start`. Значит `POST /shifts/from-template` и менеджерский overlap-guard не ловят пересечение с открытой сменой исполнителя → возможен double-booking по открытым сменам. В рамках аудита #5 НЕ чинилось (вне scope фикса `MultipleResultsFound`; фикс APIFE-1 только снял 500-краши). Требует отдельного решения: либо трактовать open-ended смену как `end_time = +infinity` в overlap-запросе, либо явно исключить из from-template. Связано с APIFE-11 (TOCTOU в PATCH overlap, `lock=False`).

- **F-03 (High, repo-side готов; ждёт prod-rollout)** — trailing-slash redirect теряет HTTPS + `/uk` (`POST https://profk.uz/uk/api/v2/auth/login/` → `307 Location: http://profk.uz/api/v2/auth/login`, воспроизведено на обоих продах 2026-07-12). Корень: edge срезает `/uk` (`rewrite ^/uk/api/(.*)$ /api/$1`) и терминирует TLS, а uvicorn стартовал без trusted-proxy (дефолт `--forwarded-allow-ips 127.0.0.1`, peer = edge-контейнер → `X-Forwarded-Proto: https` игнорился). Edge заголовки шлёт корректно — edge-side правок НЕ нужно. **Repo-side сделано:** `api`+`access-api` в `docker-compose.yml` и `docker-compose.profk.yml` получили env-driven `command` (`--proxy-headers` + `--forwarded-allow-ips $FORWARDED_ALLOW_IPS` + `--root-path $API_ROOT_PATH`); dev-safe дефолты (пусто → 127.0.0.1, без root-path); `env.example` + integration-тест `tests/api/test_proxy_root_path.py` (реальный uvicorn, 3 кейса). Выбор владельца — subnet CIDR (edge-IP не статичен, `IPAMConfig=<nil>`): profk `172.19.0.0/16`, 105 `172.20.0.0/16`. **Остаётся:** прописать `FORWARDED_ALLOW_IPS`/`API_ROOT_PATH` в `.env` обоих продов + рекрейт `api`/`access-api` + verify `curl` (307 https+/uk). `RATE_LIMIT_TRUSTED_PROXIES` (rate_limit_keys exact-match) с CIDR пока НЕ выравнивался — отдельным шагом при необходимости. См. PR-6 плана.
