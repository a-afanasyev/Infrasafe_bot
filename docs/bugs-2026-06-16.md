## S3 — UX (локализация)

### BUG-BOT-041: уведомление об одобрении заявки шлётся юзеру всегда на русском ✅ FIXED

- **Files**: `uk_management_bot/handlers/user_management.py:459` (`quick_verify_user`), `:1558` (`process_approval_comment`)
- **Эталон (корректный)**: `uk_management_bot/handlers/user_verification.py:690` — `target_lang = target_user.language or "ru"`
- **Description**: Оба хендлера строили уведомление `application_approved_restart` + кнопку `restart_bot_btn` с захардкоженным `target_lang = 'ru'` (несмотря на комментарий «Определяем язык целевого пользователя»). Сообщение уходит **конечному пользователю** (`callback.bot.send_message(chat_id=target_user.telegram_id, ...)`), а не админу. UZ-пользователь, чью заявку одобрили, получал текст и кнопку по-русски.
- **Impact**: S3 — пользователь видит чужой язык в ключевом уведомлении; функционал работает, но UX сломан для UZ-аудитории.
- **Fix**: выровнено по эталону — `target_lang = target_user.language or 'ru'` в обоих местах. `User.language` существует (`models/user.py:30`, default `"ru"`). UZ-переводы обоих ключей присутствуют в `config/locales/uz.json`.
- **Regression test**: `uk_management_bot/tests/test_bug_bot_041_approval_lang.py` — для обоих хендлеров с `target_user.language='uz'` проверяет, что текст сообщения И текст кнопки == UZ-перевод (падал на hardcoded RU, проходит после фикса).

## Заведено к разбору

### BUG-BOT-042: системный hardcoded RU в служебных/admin-рассылках 📋 DEFERRED

- **Representative files**: `uk_management_bot/handlers/user_apartment_selection.py:417` (`admin_new_apartment_request`), `handlers/auth.py:359-370` (регистрация → админу), `handlers/admin.py:236-241` (`new_request_for_duty`), `handlers/request_acceptance.py:624` (`manager_return_notification`) — всего ~13 мест с `language='ru'` в non-test коде.
- **Description**: Служебные/admin-уведомления хардкодят русский. Проект поддерживает UZ-админов, и для части ключей (напр. `admin_new_apartment_request`) **есть UZ-переводы** — т.е. локализация задумывалась, но не доведена. Это отдельный системный i18n-баг (изначально всплыл как smell #2 этого триажа).
- **Decision (owner, 2026-06-15)**: вынести в отдельный тикет, разобрать позже. Фикс — локализовать по языку получателя-админа (`admin.language`) + проверить наличие UZ-переводов всех затрагиваемых ключей. **Вне scope BUG-BOT-041.**
- **Severity**: S3 (UX для UZ-админов).
