## Найденные баги

### BUG-1: Категории и срочность в manager view показывают internal keys (P0)

**Где**: Панель менеджера → Новые/Активные заявки → список и детали
**Ожидалось**: Локализованные названия: "Сантехника", "Электрика", "Обычная"
**Факт**: Internal keys: `plumbing`, `electricity`, `low`
**Примеры**:
- Список: `🆕 #260322-003 • plumbing • Дом: Yangi Olmazor, 14V`
- Детали: `📂 Категория: plumbing`, `⚡ Срочность: low`
**Примечание**: Старые записи (legacy) показывают русские названия корректно: `#260321-001 • Электрика`
**Причина**: Новый код сохраняет internal key (category_plumbing → plumbing), но manager view не проходит через `resolve_category_key` / `get_category_display`

### BUG-2: Исполнитель не видит назначенные ему заявки (P0)

**Где**: Роль executor → "🛠 Активные заявки"
**Ожидалось**: Список назначенных заявок (как минимум #260322-003, executor_id=43)
**Факт**: "Пока нет назначенных вам заявок"
**Дополнительно**: Текст дублируется: "Пока нет назначенных вам заявок.Выберите заявку для просмотра деталей:" (нет пробела)
**Подтверждено**: В БД `executor_id=43` для request_number='260322-003', user id=43 — совпадает
**Причина**: Вероятно, фильтрация по специализации или другой критерий отсекает заявки

### BUG-3: /start не сбрасывает FSM (P1)

**Где**: В процессе создания заявки (этап выбора категории) → отправка /start
**Ожидалось**: FSM сброшен, возврат в главное меню
**Факт**: "⚠️ Пожалуйста, используйте кнопки для выбора категории" — пользователь застрял в FSM
**Влияние**: Пользователь не может выйти из FSM командой /start, только через ❌ Отмена

### BUG-4: Шаблон {start_time} не подставляется в тексте смены (P1)

**Где**: Роль executor → 🔄 Смена → ℹ️ Моя смена
**Ожидалось**: "🟢 Активная смена с 08:00"
**Факт**: "🟢 Активная смена с {start_time}" — необработанный шаблон
**Причина**: `get_text()` не получает параметр start_time или format string не применяется

### BUG-5: "📋 Мои смены" недоступна для executor (P1)

**Где**: Роль executor → "📋 Мои смены"
**Ожидалось**: Список смен исполнителя
**Факт**: "🚫 У вас нет прав доступа к этой функции."
**Влияние**: Кнопка показана но функция недоступна — нарушение UX

### BUG-6: Нет исполнителей при назначении по специализации (P1)

**Где**: Менеджер → заявка → В работу → Конкретному исполнителю
**Ожидалось**: Список исполнителей
**Факт**: "Нет доступных исполнителей", specialization: `other`
**Причина**: Категория `plumbing` маппится на specialization `other`, а не `plumbing`. Нет исполнителей со специализацией `other`
