# Полный план тестирования UK Management Bot + Web Dashboard

> **Цель:** Полноценное функциональное тестирование Telegram-бота и веб-панели, сверка данных между ними.
> **Бот:** @infrasafebot
> **Web:** Dashboard (admin/manager), Resident Board, TWA Mini App
> **Роли:** Заявитель (applicant), Исполнитель (executor), Менеджер (manager)

---

## Фаза 1: Бот — Базовые сценарии (Applicant)

### 1.1 Авторизация и /start
- [ ] `/start` — приветствие, главное меню с кнопками
- [ ] `/start` повторный — FSM сброс, возврат в меню
- [ ] `/help` — полный текст справки
- [ ] `/menu` — главное меню

### 1.2 Профиль
- [ ] "👤 Профиль" — отображение имени, телефона, ролей, адресов, языка
- [ ] "✏️ Редактировать профиль" → смена языка на UZ
- [ ] Проверить что интерфейс переключился на узбекский
- [ ] Вернуть язык на RU
- [ ] Редактирование телефона
- [ ] Редактирование имени

### 1.3 Создание заявки (полный цикл по каждой категории)
- [ ] **Электрика** → адрес (квартира) → описание → срочность (Обычная) → без медиа → подтверждение → заявка создана
- [ ] **Сантехника** → адрес (дом) → описание → срочность (Средняя) → без медиа → подтверждение
- [ ] **Уборка** → адрес (двор) → описание → срочность (Срочная) → без медиа → подтверждение
- [ ] **Благоустройство** → ручной ввод адреса → описание → срочность (Критическая) → без медиа → подтверждение
- [ ] **Безопасность** → адрес → описание → без медиа → подтверждение
- [ ] **Отопление** → адрес → описание → без медиа → подтверждение
- [ ] **Лифт** → адрес → описание → без медиа → подтверждение
- [ ] **Интернет/ТВ** → адрес → описание → без медиа → подтверждение
- [ ] Создание заявки с фото (отправить 1 фото)
- [ ] Создание заявки с несколькими медиа (2-3 фото)
- [ ] Отмена создания на этапе выбора категории (❌ Отмена)
- [ ] Отмена создания на этапе ввода описания
- [ ] `/start` во время создания заявки — FSM сброс, возврат в меню

### 1.4 Мои заявки
- [ ] "📋 Мои заявки" — список заявок пользователя
- [ ] Открыть заявку — детальная информация (номер, категория, статус, адрес, описание, дата)
- [ ] Пагинация (если > 10 заявок)
- [ ] Фильтры по статусу (если есть)

### 1.5 Приёмка заявок
- [ ] "✅ Ожидают приёмки" — список завершённых заявок
- [ ] Принять заявку с оценкой (5 звёзд)
- [ ] Принять заявку с низкой оценкой (1-2 звезды)
- [ ] Вернуть заявку на доработку с комментарием

### 1.6 Переключение ролей
- [ ] "🔀 Выбрать роль" — отображение доступных ролей с галочкой на активной
- [ ] Переключение applicant → executor — меню меняется
- [ ] Переключение executor → manager — меню меняется (появляется "🔧 Админ панель")
- [ ] Переключение manager → applicant — меню меняется (нет админ кнопок)

## Фаза 4: Бот — Edge Cases и негативные сценарии

### 4.1 FSM Reset
- [ ] `/start` из FSM создания заявки (RequestStates)
- [ ] `/start` из FSM редактирования профиля (ProfileEditingStates)
- [ ] `/start` из FSM управления адресами (YardManagementStates)
- [ ] `/start` из FSM управления сменами (ShiftManagementStates)
- [ ] `/start` из FSM верификации (UserVerificationStates)
- [ ] `/start` из FSM передачи смен (ShiftTransferStates)

### 4.2 Некорректный ввод
- [ ] Ввод текста вместо выбора кнопки на этапе выбора категории
- [ ] Пустое описание заявки (пробелы)
- [ ] Слишком длинное описание (> 4096 символов)
- [ ] Отправка стикера вместо текста описания
- [ ] Отправка голосового сообщения

### 4.3 Параллельные действия
- [ ] Быстрое двойное нажатие на inline-кнопку
- [ ] Нажатие на кнопки старого сообщения после /start
- [ ] Создание заявки пока менеджер назначает другую

### 4.4 Права доступа
- [ ] Заявитель пытается зайти в "📋 Мои смены" — отказ @require_role
- [ ] Заявитель пытается зайти в "🔧 Админ панель" — нет кнопки
- [ ] Исполнитель пытается зайти в "🔧 Админ панель" — нет кнопки
- [ ] `/admin` от обычного пользователя — отказ

## Фаза 6: Сверка данных Бот ↔ Web

### 6.1 Заявки
- [ ] Создать заявку в боте → проверить появление на Kanban в вебе
- [ ] Создать заявку через CallCenter в вебе → проверить в "📋 Мои заявки" бота (для заявителя)
- [ ] Изменить статус в вебе → проверить статус в боте
- [ ] Изменить статус в боте → проверить Kanban в вебе
- [ ] Назначить исполнителя в вебе → проверить в "🛠 Активные заявки" бота (executor)
- [ ] Назначить исполнителя в боте → проверить executor_name в вебе
- [ ] Добавить комментарий в вебе → проверить в боте
- [ ] Номер заявки совпадает (формат #YYMMDD-NNN)
- [ ] Категория, описание, адрес, срочность — идентичны
- [ ] Медиа-файлы: фото отправленное в боте видно в вебе

### 6.2 Сотрудники
- [ ] Создать сотрудника в вебе → проверить появление в боте ("👷 Управление сотрудниками")
- [ ] Создать приглашение в боте → проверить в вебе (InviteTab)
- [ ] Одобрить сотрудника в вебе → проверить роли в боте
- [ ] Заблокировать в вебе → бот отказывает в доступе
- [ ] Специализация в вебе = специализация в боте
- [ ] Статус (verified/pending/rejected) синхронизирован

### 6.3 Смены
- [ ] Создать смену в вебе → проверить в "📋 Мои смены" бота (executor)

- [ ] Сдать смену в боте → проверить статус "completed" в вебе
- [ ] Покрытие (coverage_pct) совпадает
- [ ] Передача смены в боте → отображается в вебе (transfers panel)

### 6.4 Адреса
- [ ] Создать двор в боте → проверить на странице Addresses в вебе
- [ ] Создать здание в вебе → проверить в боте ("📍 Справочник адресов")
- [ ] Создать квартиру в вебе → проверить доступность при создании заявки в боте
- [ ] Модерация: одобрить жителя в вебе → проверить привязку в боте
- [ ] Модерация: одобрить в боте → проверить в вебе
- [ ] Статистика (дворы/здания/квартиры/жители) совпадает

### 6.5 Аналитика
- [ ] Счётчик заявок в вебе (total_requests) = количество в боте
- [ ] ТОП исполнителей в вебе корректен после завершения заявок в боте
- [ ] Среднее время решения соответствует фактическим данным
- [ ] Распределение по категориям = сумме заявок созданных в боте

### 6.6 Шаблоны
- [ ] Создать шаблон в вебе → создать смену из него в боте

### 6.7 Уведомления
