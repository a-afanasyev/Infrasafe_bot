## 1. KANBAN-ДОСКА: Drag-and-Drop

### [ИСПРАВЛЕНО 2026-03-12] 1.1--1.4 TransitionModal

Проблемы 1.1--1.4 устранены внедрением компонента `TransitionModal` (`frontend/src/components/kanban/TransitionModal.tsx`).

При drag-and-drop в статусы "В работе", "Закуп", "Уточнение", "Выполнена", "Исполнено" теперь открывается контекстно-зависимое модальное окно:

| Целевой статус | Что запрашивает модальное окно | Передаваемое поле |
|---|---|---|
| В работе | Выбор исполнителя (из списка верифицированных + "Дежурный") | `executor_id` |
| Закуп | Описание необходимых материалов (textarea, обязательно) | `requested_materials` |
| Уточнение | Вопрос к жителю (textarea, обязательно) | `notes` |
| Выполнена | Отчёт о выполнении (textarea, обязательно) | `completion_report` |
| Исполнено | Подтверждение (текстовое предупреждение, без доп. полей) | -- |

Если заявка уже имеет `executor_id` при переводе "В работе", модальное окно пропускается.

### 1.5 Перевод в "Отменена" -- нет причины отмены

| Параметр | Значение |
|---|---|
| **Файл** | `frontend/src/components/kanban/KanbanBoard.tsx` |
| **Что сейчас** | State machine разрешает переход в "Отменена" из большинства статусов (кроме "Принято" и "Отменена"). Переход происходит через drag-and-drop без модального окна (статус "Отменена" не входит в `MODAL_STATUSES`). |
| **Что должно быть** | Модальное окно с запросом причины отмены при drag-and-drop в "Отменена". |
| **Серьезность** | **СРЕДНЯЯ** -- заявки можно отменить, но без фиксации причины. |

### [ИСПРАВЛЕНО 2026-03-12] 1.6 Подтверждение при drag-and-drop

Подтверждение реализовано через `TransitionModal` для ключевых переходов. Для остальных переходов (например, "Закуп" -> "В работе") по-прежнему используется оптимистичное обновление без подтверждения.

### [ИСПРАВЛЕНО 2026-03-12] 1.7 State Machine

State machine на фронтенде теперь зеркалирует бэкенд `_REQUEST_VALID_TRANSITIONS`:

```typescript
const VALID_TRANSITIONS: Record<string, Set<string>> = {
  'Новая':     new Set(['В работе', 'Закуп', 'Уточнение', 'Отменена']),
  'В работе':  new Set(['Закуп', 'Уточнение', 'Выполнена', 'Отменена']),
  'Закуп':     new Set(['В работе', 'Уточнение', 'Отменена']),
  'Уточнение': new Set(['В работе', 'Отменена']),
  'Выполнена': new Set(['Исполнено', 'В работе']),
  'Исполнено': new Set(['Принято', 'В работе']),
  'Принято':   new Set(),
  'Отменена':  new Set(),
}
```

Функция `isTransitionAllowed` экспортируется из `KanbanBoard.tsx` и используется в `KanbanColumn.tsx` для визуальной индикации допустимых зон сброса.

### [ИСПРАВЛЕНО] 2.6 RequestCard в API теперь отдает executor_name

Схема `RequestCard` в `schemas.py` содержит поле `executor_name: Optional[str] = None`. API-роутер заполняет его через JOIN с таблицей `users`:

```python
def _make_request_card(req, exec_user=None) -> RequestCard:
    card = RequestCard.model_validate(req)
    card.executor_name = _format_executor_name(exec_user)
    return card
```

Также добавлены поля `completion_report`, `notes`, `requested_materials`, `return_reason`, `source` в `RequestCard`.

## 8. ОТКРЫТЫЕ ВОПРОСЫ К ВЛАДЕЛЬЦУ ПРОДУКТА

### [ЗАКРЫТ] Q1: State Machine статусов
**Решение**: Реализована единая state machine в бэкенде (`_REQUEST_VALID_TRANSITIONS` в `api/requests/router.py`) и зеркально на фронтенде (`VALID_TRANSITIONS` в `KanbanBoard.tsx`). "Шаг назад" разрешён: "Выполнена" -> "В работе" и "Исполнено" -> "В работе".

### [ЗАКРЫТ] Q2: Статус "Исполнено" vs "Выполнена"
**Решение**: "Выполнена" = исполнитель отчитался, "Исполнено" = менеджер подтвердил (drag-and-drop через TransitionModal). Переход: "Выполнена" -> "Исполнено" с подтверждением.

### Q3: Обязательность адреса (открыт)
**Контекст**: Ни TWA, ни CallCenter не требуют ввода адреса, хотя поле есть в модели. В API `CreateRequestBody` поля `apartment_id` и `address` -- optional.
**Вопрос**: Должен ли адрес быть обязательным? Берётся ли он автоматически из профиля жителя?

### [ЗАКРЫТ] Q4: Закупки -- кто заполняет список материалов?
**Решение**: На фронтенде материалы заполняет менеджер через TransitionModal при drag-and-drop в "Закуп" (поле `requested_materials`). В боте материалы запрашивает исполнитель.

### [ЗАКРЫТ] Q5: Рейтинг из TWA
**Решение**: TWA теперь передаёт рейтинг в PATCH `/api/v2/requests/{number}` с полем `rating`.
