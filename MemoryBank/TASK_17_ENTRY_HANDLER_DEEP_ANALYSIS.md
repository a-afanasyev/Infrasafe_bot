### 3. Анализ работы фильтров aiogram

#### Как работают фильтры в aiogram:

**КРИТИЧЕСКИЙ МОМЕНТ**: Фильтры в aiogram вычисляются **при регистрации handler'а** (во время импорта модуля), а не во время выполнения.

**Пример из проекта** (`admin.py:1118`):
```python
@router.message(F.text.in_(["📋 Все исполненные", "📋 Ожидают проверки"]))
async def list_all_completed_requests(...):
```

Это работает, потому что список **статический** - создается один раз при импорте.

**Проблемный подход** (НЕ РАБОТАЕТ):
```python
@router.message(F.text.in_(get_create_request_texts()))  # ❌ Вызов функции в декораторе
```

**Почему не работает**:
- Функция `get_create_request_texts()` будет вызвана **один раз** при импорте модуля
- Если локализация изменится после импорта, фильтр не обновится
- Если `get_text()` требует инициализации БД/конфига, может быть ошибка

**Правильный подход** (РАБОТАЕТ):
```python
# На уровне модуля (вычисляется один раз при импорте)
CREATE_REQUEST_TEXTS = [
    get_text("main_menu.create_request", language="ru"),
    get_text("main_menu.create_request", language="uz")
]

@router.message(F.text.in_(CREATE_REQUEST_TEXTS))
async def start_request_creation(...):
```

**Вывод**: ✅ Нужно использовать **модульную константу**, вычисляемую при импорте.

## ⚠️ ПОТЕНЦИАЛЬНЫЕ РИСКИ И МИТИГАЦИЯ

### Риск #1: Локализация не загружена при импорте

**Вероятность**: 🟡 СРЕДНЯЯ  
**Влияние**: 🔴 ВЫСОКОЕ (handler не будет работать)

**Сценарий**:
- Модуль импортируется до инициализации локализации
- `get_text()` возвращает ключ вместо перевода
- Фильтр не срабатывает

**Митигация**:
```python
# Проверка наличия локализации
def _get_create_request_texts() -> list[str]:
    """Получить тексты кнопки создания заявки на всех языках"""
    texts = []
    for lang in ["ru", "uz"]:
        text = get_text("main_menu.create_request", language=lang)
        # Если локализация не загружена, get_text вернет ключ
        # Проверяем, что это не ключ (не содержит точки)
        if "." not in text:
            texts.append(text)
    return texts if texts else ["📝 Создать заявку"]  # Fallback на русский

CREATE_REQUEST_TEXTS = _get_create_request_texts()
```

**Альтернатива (проще)**:
```python
# Простой fallback - если локализация не работает, используем хардкод
try:
    CREATE_REQUEST_TEXTS = [
        get_text("main_menu.create_request", language="ru"),
        get_text("main_menu.create_request", language="uz")
    ]
except Exception:
    CREATE_REQUEST_TEXTS = ["📝 Создать заявку"]  # Fallback
```

**Рекомендация**: Использовать простой try-except с fallback.
