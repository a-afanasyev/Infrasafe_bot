## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### Проблема #1: requests.py - Entry Handler (BLOCKER)

**Файл**: `requests.py:380`

**Код**:
```python
@router.message(F.text == "📝 Создать заявку")
async def start_request_creation(message: Message, state: FSMContext, user_status: Optional[str] = None):
```

**Проблема**:
- Хендлер срабатывает только на точный текст "📝 Создать заявку" (русский)
- Когда главная клавиатура локализована, узбекские пользователи нажимают "📝 Ariza yaratish"
- Текст не совпадает → хендлер не срабатывает → **узбеки НЕ МОГУТ создать заявку**

**Воздействие**: 🔴 **ПОЛНЫЙ БЛОКЕР** - Узбекские пользователи не могут использовать основную функцию

**Решение**: Нужно изменить архитектуру:
- Вариант A: Использовать callback_data вместо текстовых кнопок
- Вариант B: Проверять оба языка: `F.text.in_([get_text("buttons.create_request", "ru"), get_text("buttons.create_request", "uz")])`
- Вариант C: Использовать command-based подход: `/create_request`

### Проблема #2: REQUEST_CATEGORIES - Фильтр категорий (BLOCKER)

**Файл**: `requests.py:422`

**Код**:
```python
@router.message(RequestStates.category, F.text.in_(REQUEST_CATEGORIES))
async def process_category(message: Message, state: FSMContext):
```

**Где REQUEST_CATEGORIES**:
```python
# constants.py
REQUEST_CATEGORIES = [
    "Электрика",
    "Сантехника",
    "Отопление",
    # ... все на русском
]
```

**Проблема**:
- FSM фильтр проверяет, есть ли текст сообщения в REQUEST_CATEGORIES
- Клавиатура категорий отправляет локализованный текст (например, "Elektr" для узбекского)
- Узбекский текст не в русском списке → фильтр отклоняет → **хендлер не срабатывает**
- Пользователь застревает в состоянии RequestStates.category навсегда

**Воздействие**: 🔴 **ПОЛНЫЙ БЛОКЕР** - Выбор категории сломан для узбеков

**Решение**: Нужна билингвальная система категорий:
- Использовать ID категорий (callback_data) вместо текста
- ИЛИ: Создать CATEGORIES_RU + CATEGORIES_UZ списки
- ИЛИ: Хранить канонические имена (English IDs) и маппить на отображаемые имена
