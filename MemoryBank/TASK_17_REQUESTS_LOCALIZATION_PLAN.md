2. Обеспечить корректную локализацию всех экранов заявок (детали, списки, фильтры, ошибки).

4. Подготовить базу для расширения на 3‑й язык без дополнительного рефакторинга.

## 2. Область действия

- `uk_management_bot/handlers/requests.py`

Не входит: локализация остальных handler‑файлов (будет отдельным треком) и переработка UI для ролей менеджера/исполнителя за пределами requests-модуля.

---

## 3. Технические ограничения и предпосылки

- База данных хранит значения `Request.category`/`Request.urgency` в смешанном формате (часть записей — русские строки, часть — внутренние ключи). Нужно обеспечить обратную совместимость.
- Существующие inline-клавиатуры уже используют `CALLBACK_PREFIX_CATEGORY = "category_"` с внутренними ключами — нужно распространить этот подход на фильтры и валидацию.
- Locale-файлы содержат ключи `categories.*`, `requests.*` — важно переиспользовать их, а не добавлять дубли.
- Требование пользователя: код должен оставаться понятным для джунов, без усложнённых паттернов (никакого метапрограммирования, рефлексии, наследования от абстракций ради абстракций).

## 5. Риски и способы минимизации

| Риск | Вероятность | Влияние | Митигация |
| --- | --- | --- | --- |
| Легаси-данные содержат произвольные строки категорий | Средняя | Высокое | `resolve_category_key` + fallback на оригинальный текст, логирование неизвестных значений |
| Новые locale-ключи не переведены/ошибки форматов | Средняя | Среднее | Проверка `validate_translations.py`, ручной review переводов, маркировка TODO в locale |
| Регрессия в валидаторах/сервисах (категория отклоняется) | Средняя | Высокое | Юнит-тесты на `validate_category`, smoke-тест создания заявки RU/UZ |
| Несогласованность статусов между БД и locale | Низкая | Среднее | Таблица `STATUS_KEYS` + fallback, журнал предупреждений при неизвестном статусе |

## 6. Критерии готовности

1. Категории сохраняются только как внутренние ключи; отображение всегда локализовано.

3. Валидаторы и сервисы принимают внутренние ключи и дают понятные сообщения об ошибках.
4. Docker-тесты сценариев RU+UZ пройдены; добавлены автотесты для новых helper-ов.
5. Memory Bank обновлён (issue-status, тестовый отчёт), критические пункты из `TASK_17_REQUESTS_PY_CRITICAL_ISSUES.md` закрыты.
