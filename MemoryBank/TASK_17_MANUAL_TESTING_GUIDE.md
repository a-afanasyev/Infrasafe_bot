# TASK 17: Руководство по ручному тестированию локализации

**Дата создания**: 18 ноября 2025  
**Версия**: 1.0

---

## Цель тестирования

Проверить, что все функции локализации работают корректно для русских и узбекских пользователей в Docker окружении.

---

## Подготовка к тестированию

### 1. Запуск Docker контейнера

```bash
docker compose -f docker-compose.dev.yml up -d --build
```

### 2. Проверка работы бота

Убедитесь, что бот запущен и отвечает на команды.

## Известные ограничения

1. **Ключи статусов**: Ключи локализации для статусов (`requests.status_new` и т.д.) могут отсутствовать в locale файлах. В этом случае используется fallback на русский статус.

2. **Ключи кнопок**: Некоторые ключи кнопок могут отсутствовать в locale файлах. Используется fallback на русский текст.

3. **Статусы**: Полная локализация статусов требует добавления ключей в `ru.json` и `uz.json`.
